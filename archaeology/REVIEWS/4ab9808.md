not-applicable — the C bug is a clarity/maintainability fix in
`std/filters/arr.c::wrapArrayIndices`: a pre-declared `len = 0`
variable conditionally assigned and returned, restructured into

no `src/server/database/filters/` directory and no
`wrap_array_indices` (or analog) helper anywhere in the crate. EPICS

slicing in Rust returns `&[T]` whose length is unambiguous from
`slice::len()` — the implicit-zero-via-uninitialized-variable pattern
that prompted this C clarification cannot arise.
