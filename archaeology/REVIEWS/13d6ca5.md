The C bug was that `initHookRegister(func)` appended unconditionally to the `functionList`, so a function registered twice (e.g., from a shared library loaded multiple times) would fire twice on each `initHookCall` state transition. The audit target `src/server/database/init_hooks.rs::init_hook_register` does not exist in `epics-base-rs`. There is no `init_hooks.rs` module and no `initHook` API. The closest analog is `IocApplication::register_after_init` (`src/server/ioc_app.rs:158`), which stores `Box<dyn FnOnce() + Send>` in a `Vec`. Because each closure is `FnOnce`, callers cannot accidentally cause one to fire twice — ownership is consumed on invocation. There is no global registry that deduplicates by function pointer because there is no equivalent of the multi-state `InitHookState` machine. The C-shaped double-registration bug is structurally absent.
