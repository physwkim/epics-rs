The C bug was `epicsSnprintf(buf, sizeof(buf)-1u, "%u", port)` truncating 5-digit ports (>=10000) by passing a reduced size. epics-rs has no `RSRV_SERVER_PORT` advertisement and never round-trips a port through a fixed C-style buffer; ports are formatted with `port.to_string()` which is `String`-allocated and unbounded. `cargo grep` for `RSRV_SERVER_PORT` returns zero hits in production code. The bug class is structurally absent in safe Rust string formatting.
