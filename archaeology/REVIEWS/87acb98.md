The C bug used a fixed-size `char buf[256]` in `getToken()` to copy hostname tokens, silently truncating long FQDNs. Rust ca-rs parses `EPICS_CA_ADDR_LIST` in two places: `epics-ca-rs/src/client/mod.rs::parse_addr_list` (line 2032) and `epics-ca-rs/src/server/addr_list.rs::parse_addr_list` (line 110). Both use `&str::split_whitespace()` which yields `&str` slices into the heap-allocated env value with no fixed-length buffer. Each token is passed in full to `resolve_host`/`resolve_token` for DNS resolution. No truncation possible — N/A.
