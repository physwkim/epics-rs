# Review: 951b6ac — Cygwin TCP_NODELAY include

## Verdict
not-applicable — the C bug was a missing `#include <netinet/tcp.h>` that
caused `TCP_NODELAY` to be undeclared on Cygwin. In Rust, `TcpStream::set_nodelay(bool)`
in `std::net` and `tokio::net` handles the constant internally and is
portable across all `rustc`-supported targets.

## Audit of latency-sensitive call sites
The audit recommendation was to confirm `set_nodelay(true)` is called on
every accepted/connected TCP socket:

- `server/tcp.rs:432` — `let _ = stream.set_nodelay(true);` on every
  accepted socket.
- `client/transport.rs:380` — set after `TcpStream::connect`.
- `client/search.rs:381` — set on each nameserver TCP connection.

All three call sites do call `set_nodelay(true)`, so CA control-message
latency is not subject to Nagle batching.
