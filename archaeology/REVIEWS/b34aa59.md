There is no `db_lex_routines.rs` and no `popFirstTemp`/`tempList` parser arena in base-rs. The DB/DBD parser is a hand-written recursive char-scanner in `src/server/db_loader/mod.rs` that uses local `Vec`s and propagates parse errors via `CaError::DbParseError` returned through `?`. There is no shared temporary-list state that a parse error in one phase can leave in an inconsistent state to crash the next phase. The C cascade (`popFirstTemp` returns NULL after an earlier error ‚Üí callers dereference) has no structural counterpart. Nothing to fix.
