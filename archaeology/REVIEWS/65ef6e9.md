# Review: 65ef6e9 — BSD requires SO_REUSEPORT for fanout

## Verdict
applied — same code paths as 5064931. On BSD/macOS, `SO_REUSEADDR`
alone does NOT permit two sockets to bind the same UDP port; only
`SO_REUSEPORT` does. The ca-rs server UDP responder and the in-process
repeater previously only set `SO_REUSEPORT` on macOS (responder) or
neither (repeater).

## Fix
Both `server/udp.rs::run_single_responder` and `repeater.rs::run_repeater`
now call `set_reuse_port(true)` under `#[cfg(unix)]` — covers Linux,
FreeBSD, macOS, NetBSD, OpenBSD. The cfg gate prevents the call on
Windows where `socket2::Socket::set_reuse_port` is not available.

`epics-base-rs/net/async_udp_v4.rs::bind_one_at` already does this for
the client search/beacon-monitor sockets.

## C reference
`modules/libcom/src/osi/os/posix/osdSockAddrReuse.cpp:
epicsSocketEnableAddressUseForDatagramFanout` — uses `SO_REUSEPORT` via
the `X_REUSEUDP` macro when available, falls back to `SO_REUSEADDR`.

`cargo check -p epics-ca-rs` — clean.
