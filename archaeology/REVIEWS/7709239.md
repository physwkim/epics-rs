The C bug is `dbPut()` calling `prset->put_array_info(paddr, nRequest)` without a NULL check on the optional RSET function pointer. base-rs has no `prset` / RSET indirection: record operations go through the `Record` trait (`crates/epics-base-rs/src/server/record/record_trait.rs`), whose methods are not optional — every record type provides concrete impls of `put_field`, `field_list`, `process`, etc. There is no `put_array_info` method in the trait. Array-aware behavior, when needed, is implemented inline in each record's `put_field` or via the `multi_input_links`/`multi_output_links` framework. The Rust trait dispatch is type-checked at compile time, eliminating null-callback dispatch by construction. No fix needed.
