The `db_ca.rs` and `db_convert_json.rs` targets do not exist in base-rs. The closest analog — DBR type dispatch in `crates/epics-base-rs/src/types/codec.rs` (`serialize_dbr`, `decode_dbr`) and `crates/epics-base-rs/src/types/dbr.rs` (`native_type_for_dbr`, `dbr_native_index`) — already validates `dbr_type` at the network boundary: `dbr_native_index` returns `None` for any code outside `0..=37`, and `native_type_for_dbr` lifts that to `CaError::UnsupportedType`. Dispatch uses explicit match arms (`0..=6`, `7..=13`, ...), not raw table indexing, so the C OOB pattern cannot occur. No changes needed.
