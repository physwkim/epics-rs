# Review: e88a186

## Verdict
not-applicable — structurally absent in base-rs.

## Analysis
The C bug is that `struct link::flags` in
`modules/database/src/ioc/dbStatic/link.h` was declared `short flags`
(signed), so bit-field operations on the high bit invoked signed
integer overflow UB.

`epics-base-rs` does not represent link flags as a single integer
bitfield. `DbLink` (`server/record/link.rs:46-52`) holds:

```
pub struct DbLink {
    pub record: String,
    pub field: String,
    pub policy: LinkProcessPolicy,
    pub monitor_switch: MonitorSwitch,
}
```

`LinkProcessPolicy` and `MonitorSwitch` are plain Rust enums (no `repr`
attribute, no bit operations). There is no `flags: i16/u16/u32` field
anywhere in `link.rs` — a repo-wide search in
`epics-base-rs/src/server/record/link.rs` for `flags|Flags` returns no
matches. The signed-bitfield UB pattern cannot exist.
