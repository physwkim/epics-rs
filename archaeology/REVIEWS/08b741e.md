# Review: 08b741e — Repeater fallback when spawn fails

## Verdict
not-applicable — current Rust code already does the post-fix behaviour.

## Current code
`crates/epics-ca-rs/src/repeater.rs::spawn_repeater()` (lines 308-339) tries
the external `ca-repeater-rs` binary, and if it does not exist or
`Command::spawn()` returns an error, falls through to a `std::thread::spawn`
that runs `run_repeater()` in-process. The fallback covers every spawn
failure (file-not-found, permission-denied, transient OS errors) — exactly
the behaviour the C fix introduced when it changed the condition from
`== osiSpawnDetachedProcessNoSupport` to `!= osiSpawnDetachedProcessSuccess`.

`ensure_repeater()` (lines 255-266) calls `spawn_repeater()` whenever
`try_register()` fails, then re-registers — guaranteeing that beacon
reception works even when `ca-repeater-rs` is not installed.
