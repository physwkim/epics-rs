The C bug was a hand-maintained name table inside `initHooks.c::initHookName` that drifted out of sync with the `initHookState` enum when 7 shutdown states were added. In `epics-base-rs`, no equivalent `initHookName` function or `initHookState` enum exists. The audit target `src/server/database/init_hooks.rs` is absent from the crate. The IOC lifecycle in base-rs is expressed with a generic `Vec<Box<dyn FnOnce()>>` named `after_init_hooks` on `IocApplication` (`src/server/ioc_app.rs:80`) — these are user-supplied closures invoked after `iocInit`, not a typed enum lifecycle with named string lookup. There is no shutdown-state name table that could go out of sync, so the OOB-name-string class of bug cannot occur. The whole shutdown-hook lifecycle (`initHookAtShutdown` … `initHookAfterShutdown`) is structurally absent.
