This commit reverts a09e5b5/a46bd5a's dbCa `iocInit`-blocking-wait mechanism from C-EPICS. base-rs has no `ca_link.rs`, no `dbCaTask`, and no `initOutstanding`-style blocking gate during `iocInit`. CA links in base-rs are resolved lazily via `PvDatabase::resolve_external_pv` (database/mod.rs:287), which is non-blocking — it consults registered `LinkSet`s/`ExternalPvResolver` on demand at link-read time and returns `None` if the upstream isn't ready, never waiting. Consequently the deadlock pattern this revert addresses cannot occur in base-rs, and there is no equivalent code path to remove. No fix needed.
