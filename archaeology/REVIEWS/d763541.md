# Review

## Audit
The C change adds a `getHostMinorProtocol` virtual through
`cacChannel`/`netiiu`/`tcpiiu`/`nciu`/`udpiiu`, plus a public
`ca_host_minor_protocol(chid)` API returning `tcpiiu::minorProtocolVersion`
(populated from the CA_PROTO_VERSION reply).

In `epics-ca-rs`, the server minor version was already parsed in
`client/transport.rs:706` (`server_minor_version = hdr.count`) but kept
local to the `read_loop` task and only used to choose between
`CA_PROTO_ECHO` and `CA_PROTO_READ_SYNC` for heartbeat. It was never
surfaced to the coordinator or to `CaChannel`, so callers had no way to
read it. Confirmed structurally — `host_minor_protocol` did not exist,
nor any `Channel`-side accessor analogous to `host_name`.

This is the gateway/nameserver use case called out in the analysis MD: a
gateway built on `epics-ca-rs` could not report the upstream IOC's CA
wire version to its downstream clients.

## Fix
Plumbed the version end-to-end:

1. `client/types.rs` — new `TransportEvent::ServerVersion { server_addr,
   minor_version }` variant.
2. `client/transport.rs:706` — emit `ServerVersion` alongside the existing
   local update on every CA_PROTO_VERSION frame.
3. `client/mod.rs` (coordinator) —
   - new `server_minor_version: HashMap<SocketAddr, u16>` keyed by circuit;
   - inserted on `ServerVersion`, removed on `TcpClosed`;
   - the existing `rx_addr` watchdog match now treats `ServerVersion` as a
     real RX frame so it bumps `last_rx_at` like any other server-originated
     event.
4. `client/mod.rs` — new `CoordRequest::GetHostMinorProtocol { cid, reply }`
   returning `Option<u16>` (None when the channel isn't operational or the
   circuit hasn't yet sent VERSION).
5. `client/mod.rs` — public `CaChannel::host_minor_protocol() -> Option<u16>`
   mirroring the libca shape (None ↔ `CA_UKN_MINOR_VERSION`).

Per-server keying matches libca: the version belongs to the TCP circuit
(`tcpiiu::minorProtocolVersion`), not to the channel. All channels on
the same circuit share it; reconnection re-populates from the next
VERSION frame.

C ref: `modules/ca/src/client/tcpiiu.cpp:tcpiiu::getHostMinorProtocol`,
`modules/ca/src/client/oldChannelNotify.cpp:ca_host_minor_protocol`.

## Verification
`cargo check -p epics-ca-rs` — clean build (one pre-existing unrelated
warning in `server/beacon.rs` about an unused `mut send_errors`).
