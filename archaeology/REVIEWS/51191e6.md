The shared multi-NIC UDP layer in `epics-base-rs/src/net/async_udp_v4.rs:595` already calls `set_multicast_all_v4(false)` on Linux, so anything routed through `AsyncUdpV4::bind` (CA client search, etc.) is covered. Two ca-rs UDP creators were not: `epics-ca-rs/src/server/udp.rs::run_single_responder` (CASS UDP search responder) used socket2 directly, and `epics-ca-rs/src/repeater.rs::run_repeater` used `tokio::net::UdpSocket::bind` directly. Added `#[cfg(target_os = "linux")] sock.set_multicast_all_v4(false)` to both, after switching the repeater to socket2 so the option can be set before bind (matches libcom 51191e6). Beacon and signed_beacon sockets are unicast send-only paths (`UdpSocket::bind("0.0.0.0:0")`) so they are not exposed to multicast leakage. `cargo check -p epics-ca-rs` passes.
