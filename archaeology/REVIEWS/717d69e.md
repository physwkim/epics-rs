The C bug (`dbCaRun` returning before CA links connect, so PINI processes against UDF) requires a `dbCa` worker subsystem that doesn't exist in epics-rs. CA/PVA links are recognized by the parser (`ParsedLink::Ca` in `epics-base-rs/src/server/record/link.rs:82`) and dispatched via `resolve_external_pv` (`epics-base-rs/src/server/database/mod.rs:287`), which goes through synchronous lset registries — there is no async connect+subscribe handshake and no `db_ca_run` / `db_ca_add_link_callback` machinery. There's nothing to gate on. When a real CA-link agent is added it must adopt this barrier pattern (AtomicUsize counter + Notify before transitioning to AfterIocRunning), but until then there is no fix to apply. Tracked as architectural follow-up.
