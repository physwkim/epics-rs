# Review: c23012d — Per-destination beacon send-error dedup

## Verdict
applied — `run_beacon_emitter` previously discarded send results with
`let _ = socket.send_to(&bytes, addr).await;`. A persistently
unreachable beacon destination produced no log output at all (silent
failure). The audit recommendation called for first-occurrence + change
+ recovery logging.

## Fix
`crates/epics-ca-rs/src/server/beacon.rs`:
- Added `HashMap<SocketAddr, std::io::ErrorKind>` `send_errors` per
  emitter task.
- On `Err(e)`: `tracing::warn!` only when the kind differs from the
  last recorded kind for that destination (first occurrence or change).
- On `Ok(_)`: if a previous error was recorded, emit
  `tracing::info!("CA beacon send recovered")` and clear the entry.

The state lives for the lifetime of the emitter task; addresses are
fixed at task start, so the map size is bounded.

## C reference
`modules/database/src/ioc/rsrv/online_notify.c:rsrv_online_notify_task`
— added `int *lastError` array, indexed parallel to `beaconAddrList`,
with the same dedup-on-change + recovery-log pattern.

## Build
`cargo check -p epics-ca-rs` — clean.
