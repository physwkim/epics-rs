# Review

## Verdict
**not-applicable** — base-rs implements no channel-filter plugins and no
`db_field_log` struct.

## Evidence
- `audit_targets.file = src/server/database/filters/ts.rs` does not exist
  (no `filters/` subdirectory under `src/server/database/`).
- Repository-wide grep for `db_field_log`, `DbFieldLog`, `ts_filter`, `TsFilter`,
  `replace_fl_value`, `channelRegisterPost` returns no hits in the base-rs source
  tree.
- The Rust pipeline reads source-record state directly via
  `links::read_link_with_alarm` (`src/server/database/links.rs:53`) and does not
  manufacture an in-memory field-log carrier with a destructor callback. There
  is no equivalent of `pfl->u.r.dtor` vs `pfl->dtor` confusion to inherit.

## Decision
No struct-layout migration to chase, no `dtor` callback registration site to
move. If the channel-filter plumbing (incl. `ts`/`dbnd`/`arr`/`sync` filters
and `channelRegisterPost`) is ported later, place the destructor at the top
level of the field-log carrier so cleanup is invoked through the same field
the runtime uses.
