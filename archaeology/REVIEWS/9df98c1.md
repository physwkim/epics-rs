# Review

## Verdict
**not-applicable** — base-rs has no log-client TCP reconnect loop to audit.

## Evidence
- `audit_targets.file = src/log/log_client.rs` does not exist.
- No `log_client`, `LogClient`, `reconnect_loop`, or equivalent TCP-backed
  log-shipping worker exists in `crates/epics-base-rs/`.
- `src/runtime/log.rs` is the entire logging surface: synchronous
  `eprintln!`-backed macros with no socket, no buffer, no connect/flush states
  and therefore no dual-arm select bug to inherit.

## Decision
The C bug — `if (isConn) flush(); else connect();` causing flush to be skipped
on the iteration that connects — has no Rust counterpart because the structural
preconditions (a persistent retry loop with separate connect/flush arms over a
buffered TCP log channel) are absent. Re-audit if/when an async log-shipping
client is introduced; the fix pattern is `connect.await?; flush().await?` so
flush always runs after a successful connect.
