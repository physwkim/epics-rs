The decimate and sync channel filter plugins do not exist in base-rs. C `dbDbLink` invokes a chain of filter plugins (`dbChannelRunPreChain` / `RunPostChain`) that can rewrite or drop monitor events; base-rs has no such chain. Monitor delivery in `RecordInstance::notify_subscribers_with_event` (record_instance.rs:1395-1450) is a single direct mask-intersection: `if sub_mask.intersects(posting_mask)`. There is no per-subscription stateful counter or gate that could swallow a `DBE_PROPERTY` event, so the bug class (filter dropping property updates while gating value updates) cannot exist. Nothing to fix.
