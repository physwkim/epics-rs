There is no `db_db_get_value` and no `FieldLog`/`field_log`/`db_field_log` type in base-rs. The C `dbDbLink::dbDbGetValue` flow (allocate field-log → run pre-chain → fetch value → run post-chain → delete field-log) does not exist; `links.rs::read_link_value` directly calls `get_pv`/`get_record` and returns an `EpicsValue` with no filter chain in between. There is no pre/post chain that could swap a heap pointer, so the "stack-local vs heap-owned field log lifetime" question is moot. Nothing to fix.
