# Review — 410921b: getifaddrs vs SIOCGIFCONF (broadcast discovery)

## Audit target resolution
The metadata named `src/client/net_intf.rs::discover_broadcast_addresses`,
but no such file exists. The actual ca-rs broadcast-enumeration logic lives
in `crates/epics-ca-rs/src/server/addr_list.rs::discover_broadcast_addrs`
(used both by the server and by the client search path in
`src/client/mod.rs:2066`). A second `if-addrs` consumer is
`src/discovery/mdns.rs::announce` (mDNS announce IP set).

## Audit checklist (vs. recommendation in commit MD)

1. **Uses `getifaddrs` (not `SIOCGIFCONF`)** — OK.
   `discover_broadcast_addrs` calls `if_addrs::get_if_addrs()`, which on
   POSIX (`if-addrs` 0.13.4 `src/lib.rs:183`) wraps `getifaddrs(3)`
   directly. Same on Windows via `GetAdaptersAddresses`.

2. **`IFF_BROADCAST` filter preserved** — OK.
   `if-addrs` only populates `Ifv4Addr::broadcast = Some(_)` when
   `ifa_flags & IFF_BROADCAST` (== 2) is set
   (`if-addrs/src/lib.rs:195-197`). Our code matches on
   `Some(b) = v4.broadcast`, so non-broadcast NICs are skipped — same
   behaviour as `osdNetIfAddrs.c:130` (`if (ifa->ifa_flags & IFF_BROADCAST)`).

3. **Loopback exclusion** — OK.
   `iface.is_loopback()` short-circuits before reading the addr
   (`addr_list.rs:163-165`), matching `osdNetIfAddrs.c:108`
   (`IFF_LOOPBACK`).

4. **Local-address `Once` cache** — N/A.
   ca-rs has no `osiLocalAddr`-style API surface (no consumer queries a
   single "primary local IPv4"); the cache need does not arise. mDNS
   announcement enumerates fresh per-call but only on register, so caching
   is unnecessary.

## Parity gap found and fixed
`osdNetIfAddrs.c:133` discards an interface when
`baddr.ia.sin_addr.s_addr != INADDR_ANY` is **false** — i.e. it skips
broadcasts that resolve to `0.0.0.0`. The Rust port did not. While
`if-addrs` rarely returns `Some(0.0.0.0)`, a degenerate point-to-point
or freshly-configured interface can produce one, and propagating
`0.0.0.0:5064` to the search address list is wrong (the kernel either
errors `EADDRNOTAVAIL` on `sendto`, or — worse on some BSDs — sends to
the limited broadcast).

Fix: added `if b.is_unspecified() { continue; }` guard inside the
`Some(b) = v4.broadcast` arm, with a comment citing the C reference.

## Parity gap NOT fixed (out of scope, >50-line cost)
- **`IFF_UP` filter** (`osdNetIfAddrs.c:100-103`): `if-addrs` 0.13.4 does
  not expose interface flags. Adding it would require either swapping to
  `pnet_datalink`/`netdev` or a direct `nix::ifaddrs` call plus per-OS
  flag handling — well over 50 LOC. In practice on POSIX `getifaddrs(3)`
  rarely returns down interfaces with assigned IPs, so the user-visible
  impact is small.
