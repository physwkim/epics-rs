# Review

## Verdict
**not-applicable** — base-rs has no `errlog` worker thread/task to audit.

## Evidence
- `audit_targets.file = src/log/errlog.rs` does not exist.
- The only logging surface is `crates/epics-base-rs/src/runtime/log.rs`, which
  defines four trivial `eprintln!` macros (`rt_debug!`, `rt_info!`, `rt_warn!`,
  `rt_error!`). There is no ring buffer, no worker task, no `atExit` flag, no
  `flush()` API, and no shutdown drain logic to get wrong.
- A repository-wide search for `errlog`, `errlog_thread`, `atExit`, `nFlushers`,
  `waitForSeq` finds only a single comment reference inside
  `src/server/record/record_instance.rs:747` describing C EPICS semantics — no
  Rust port of the worker exists.

## Decision
Macros write synchronously to stderr, so the "queued messages dropped at
shutdown" + "flush callers hang forever" failure modes from
`libcom/src/error/errlog.c` cannot manifest in this codebase. No fix to apply.
If a buffered async errlog is added later, the C-7448a8b drain-then-exit pattern
should be reapplied.
