The `db_ca_sync` / `db_ca_task` work-queue subsystem does not exist in base-rs. There is no `db_ca.rs` and no CA-link worker channel that needs sync-sentinel handling. The Rust rewrite uses direct `process_record_with_links` dispatch for CP links; if a future refactor introduces a tokio mpsc-based CA link worker, it must implement `Sync` as a queue-empty oneshot per the C fix.
