# Review: 5b37663 — NULL guard for aToIPAddr

## Verdict
not-applicable — the crash class does not exist in the Rust port.

## Why
`aToIPAddr()` was a C function that dereferenced a possibly-NULL pointer.
In `epics-ca-rs` the equivalent parsers (`server/addr_list.rs::parse_addr_list`,
`parse_ipv4_list`, `resolve_token`) take `&str`, not `Option<&str>` or a raw
pointer — a NULL value cannot reach them.

Every call site that derives the input from environment variables uses
`epics_base_rs::runtime::env::get(...)`, which returns `Option<String>`,
and is consumed via `if let Some(list) = ...` — verified in
`server/addr_list.rs:46,58,70,72,94,98` and `client/mod.rs:2041`,
`repeater.rs:160-161`. Unset environment yields `None` and is handled
gracefully (the parser is simply not called).

Empty strings flow through `split_whitespace().filter_map(...)` and are
benign (zero entries produced).
