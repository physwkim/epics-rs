The audit targets `callbackInit`/`callbackStop`/`callbackCleanup` in `src/server/database/callback.rs` do not exist in `epics-base-rs`. There is no callback subsystem at all in the rewrite — no `cbCtl`/`cbState`/`callbackIsInit` flags, no per-priority worker queues, no `callbackTask` shutdown handshake. The IOC lifecycle in base-rs is managed via `IocApp::run`/`IocBuilder` and tokio task supervision (`runtime/supervise.rs`), which uses tokio's cancellation tokens / `JoinHandle::abort` for shutdown. Atomic state transitions are unnecessary because there are no parallel C-style worker threads racing on a shared enum — task lifecycle is driven by the tokio runtime, which provides its own synchronization. The init/stop/cleanup race the C atomic-CAS fix targets does not arise.
