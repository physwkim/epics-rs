The audit target `src/server/database/event.rs::db_flush_extra_labor_event` does not exist in `epics-base-rs`. There is no `event.rs`, no `extra_labor`/`extraLaborBusy`/`extralabor_sub` mechanism, no `pflush_seq` counter, and no `event_task` worker loop. The rewrite has no separate event-delivery worker that needs flush-synchronization — monitor delivery is push-based via per-subscriber `mpsc::Sender`s pushed inline from record-processing code (see `RecordInstance::notify_field` / `notify_from_snapshot`). There is no busy-wait sleep loop and no labor-pending flag to race against. Drop-based subscription cancellation already provides the synchronization the C `db_sync_event` helper was introduced to give callers.
