The C `dbr_size_n(TYPE, COUNT)` empty-array bug was already absorbed in
round-8 P-1 (commit d5253b4) at the corresponding Rust site.
`crates/epics-base-rs/src/types/value.rs::EpicsValue::from_bytes_array`
now branches on `count == 0` to return the typed empty-array variant
(`StringArray(Vec::new())`, `ShortArray(Vec::new())`, ..., `CharArray(Vec::new())`)
before any value-byte read. `count == 1` falls through to the scalar
decoder; `count >= 2` hits the per-type loops with bounded `cap_for(elem_size)`.
A separate empty-array UDF/INVALID alarm overlay was added in F6
(674dd37) for the bridge NTScalar conversion path. The original CAGET
zero-element waveform garbage-read symptom is no longer reachable in
ca-rs. No further action required.
