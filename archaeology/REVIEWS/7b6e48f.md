The C bug is `casw.cpp` using `epicsTime::getMonotonic()` for both
`programBeginTime` and beacon arrival times where wall-clock semantics
were required. epics-rs has no `casw` binary
(`crates/epics-ca-rs/src/bin/` lists only ca-admin/ca-lint/ca-repeater/
ca-replay/ca-soak/caget/cainfo/camonitor/caput/softioc); the audit target
`src/bin/casw.rs::main` does not exist. The closest live code,
`crates/epics-ca-rs/src/client/beacon_monitor.rs`, only uses
`std::time::Instant` for inter-beacon period estimation and "last_seen"
gap detection (correctly monotonic for elapsed-time math); it never
correlates against a wall-clock value embedded in the beacon payload
because the CA beacon wire format does not carry one. No fix needed.
