The C bug is in `dbConvertJSON.c::dbPutConvertJSON`: an empty input string (`jlen == 0`) was passed straight to `yajl_alloc`/`yajl_parse`, which returns a parse error. An empty JSON value should be treated as a zero-element array (`*pnRequest = 0; return 0;`).

`crates/epics-base-rs` has no JSON-link conversion path. Grep for `json|yajl|JsonLink|put_convert_json|serde_json` across the crate returns no hits — there is no `db_convert_json.rs`, no `dbConstLink` JSON form, and no parser dependency that could choke on an empty buffer. Constant input links in base-rs (`server/database/links.rs`, `server/record/link.rs`) carry parsed scalar/array values directly; they are not re-parsed from a JSON source string at link-eval time. The defect is structurally absent.
