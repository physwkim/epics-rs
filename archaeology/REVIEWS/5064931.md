# Review: 5064931 — Both REUSEPORT and REUSEADDR for fanout sockets

## Verdict
applied — `server/udp.rs::run_single_responder` previously only enabled
`set_reuse_port(true)` on macOS, leaving Linux with `SO_REUSEADDR` alone.
On Linux the two flags form different reuse classes — a socket holding
the port with `SO_REUSEADDR` will not let a `SO_REUSEPORT`-only socket
rebind, and vice-versa. The CA server and an in-process caRepeater
sharing UDP 5065 in development setups would EADDRINUSE.

## Fix
`server/udp.rs:68-79` — refactored the bind to:
```
#[cfg(not(windows))]
{
    sock.set_reuse_address(true)?;
    #[cfg(unix)]
    sock.set_reuse_port(true)?;
}
```
Same idiom added to `repeater.rs::run_repeater` for the listen-on-5065
fanout socket.

`epics-base-rs/net/async_udp_v4.rs::bind_one_at` already set both flags
on Unix; only the Windows-gating part was changed there (see 19146a5).

## C reference
`modules/libcom/src/osi/os/posix/osdSockAddrReuse.cpp:
epicsSocketEnableAddressUseForDatagramFanout` — sets both
`SO_REUSEPORT` and `SO_REUSEADDR` unconditionally on POSIX.
