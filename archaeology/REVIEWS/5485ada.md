# Review: 5485ada

## Verdict
not-applicable — structurally absent in base-rs.

## Analysis
The C bug is that `epicsNAN`/`epicsINF` were declared as mutable
`extern float` globals (no `const`), allowing accidental mutation of
the sentinels.

`epics-base-rs` defines no NaN/Inf sentinel globals. A repo-wide
search for `static mut <float>`, `epicsNAN`, `epicsINF`, `EPICS_NAN`,
`EPICS_INF` in the base-rs `src/` tree returns only one hit — a code
comment in `server/records/sel.rs:58` referring to the C-side
behavior. No Rust `static mut` floats exist.

Any code needing NaN/Inf uses `f32::NAN`, `f32::INFINITY`,
`f64::NAN`, `f64::INFINITY` directly (built-in immutable consts), so
the mutable-sentinel hazard cannot exist.
