The Rust ca-repeater (`epics-ca-rs/src/repeater.rs::run_repeater`) bound `0.0.0.0:5065` and never called `IP_ADD_MEMBERSHIP`, exactly mirroring the C bug. Added `join_beacon_multicast_groups()` which reads `EPICS_CAS_BEACON_ADDR_LIST` (falling back to `EPICS_CA_ADDR_LIST`), parses each token, strips an optional `:port`, and calls `socket2::Socket::join_multicast_v4(addr, INADDR_ANY)` for every address satisfying `Ipv4Addr::is_multicast()` (224.0.0.0/4). Failures are logged via `tracing::warn` but non-fatal — unicast/broadcast beacons keep flowing. The join happens after `bind()` and before the socket is converted to `tokio::net::UdpSocket`, matching the C ordering. `cargo check -p epics-ca-rs` passes.
