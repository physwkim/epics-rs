ca-rs has no `udpiiu.rs` (the UDP exception path is folded into `client/transport.rs`'s TCP read loop, since the CA UDP path in this crate carries only search/beacon traffic). The closest equivalent of `udpiiu.cpp::exceptionRespAction` is the `CA_PROTO_ERROR` branch in the TCP read loop at `crates/epics-ca-rs/src/client/transport.rs:768-789`, which previously printed the server error via raw `eprintln!("CA server error: cmd={:?} msg={}", ...)`. C ref `modules/ca/src/client/udpiiu.cpp:exceptionRespAction` was changed in commit a352865 to route the prefix through `ERL_ERROR` so the operator sees a coloured `Error:` on a TTY and a structured prefix in log files. The Rust equivalent is `tracing::error!`, which honours the configured subscriber's formatting (ANSI-aware on TTY, plain in pipes, structured fields downstream). Replaced the bare `eprintln!` with a `tracing::error!` that emits the same fields plus `server` for context, matching the spirit of the C fix.
