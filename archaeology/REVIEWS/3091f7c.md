There is no `int64in` (or `int64out`) record implementation in base-rs (`crates/epics-base-rs/src/server/records/` contains only ai/ao/bi/bo/longin/longout/etc.). The DELTA truncation bug therefore has no current target. When int64in/int64out are added, the monitor delta must be computed as `i64.abs_diff()` / `u64`, never cast through `u32`.
